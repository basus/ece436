Programming for the Web

Using HTML, JavaScript and CSS

What is XML?

  • eXtensible Markup Language
  • Set of rules for encoding documents in machine-readable form
  • Plain text data format with support for Unicode
  • XML documents must be well-formed – satisfy the syntax rules in the XML specification
  • XML allows you to define your own markup languages for specific applications
  • XML markup languages include HTML, RSS, MathML and DocBook

An XML example

              <tag attribute="value">
              
                  Some Content

                  <nested>
                      Some more content
                  </nested>

                  <empty-tag />
              </tag>
  • XML data has a tree-like structure
  • XML parsers check for validity and well-formedness against a given specification

What is HTML?

  • HyperText Markup Language
  • It is a language for structuring content for the World Wide Web
  • Provides structural meaning for text by providing headings, paragraphs, lists, links, etc
  • Allows links to other resources on the Internet and embedding of images, audio and video
  • HTML5 is a formulation of HTML as an XML document type
              <!DOCTYPE html>

HTML is meant for structure and content

CSS is for styling a structured document


  • Use HTML to structure your document (headers, paragraphs, lists etc).
  • Use CSS to describe how elements look (fonts, colors, weight).

What is CSS?

  • Cascading Style Sheets
  • Select HTML elements
  • Apply styling information to selected HTML elements
  • Most commonly used for HTML, but can be applied to any XML format

What is JavaScript?

  • Dynamic, object-oriented programming language
  • Provides variables, objects, arrays and functions (but not classes)
  • Syntax is similar to C and Java, but has little in common with them
  • Designed to be used embedded in an environment such as a browser or widget engine
  • Can be interact with HTML pages via the Document Object Model

A simple JavaScript example

function factorial(n) {

    if (n == 0) {
        return 1;
    } else {
        return n * factorial(n - 1)
    }
}
        
var fact0 = factorial(0);
var fact32 = factorial(32);

Building a web application

  1. Create an HTML page (with optionally CSS styling)
  2. Use JavaScript to send information requests via HTTP
  3. Have a server to listen to HTTP requests and send replies in XML format
  4. Use JavaScript to receive the reply and extract relevant information
  5. Use JavaScript to manipulate the DOM to present the extracted information

XML HTTP Request

var xmlHttp = new XMLHttpRequest();

xmlHttp.onreadystatechange = some_function;
xmlHttp.open("GET", URL, true);
xmlHttp.send(null);


function some_function {

    Do something with data in xmlHttp.responseText

}
  • Server responds to requests with text data — generally in XML.
  • Responses may be generated dynamically.
  • Server can be written in any language — PHP, Python, Ruby, Java, etc.

XHR Example

var xml;
function request() {
    xml = new XMLHttpRequest();
    xml.onreadystatechange = respond;
    xml.open("GET", "test.xml", true);
    xml.send(null);
}

function respond() {
    if (xml.readyState == 4 && xml.status == 200 ) {
        var em = document.getElementById("xhr-ex");
        var hello = document.createElement("p");
        var hello_text = document.createTextNode(xml.responseText);
        hello.appendChild(hello_text);
        em.appendChild(hello);
    } 
};

In Summary

  1. Use HTML to create structured documents
  2. Use CSS to apply visual styles to an HTML document
  3. Use JavaScript (with libraries) to manipulate the DOM
  4. Use JavaScript and XML HTTP Request to make requests to a server and update the web page
  5. Use a server written in any programming language to send text data back to a web page
